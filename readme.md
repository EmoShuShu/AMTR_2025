# 万智牌比赛规则精解_2025_7_26

## 📖 项目简介

MTR精解（Annotated MTR）是对MTR逐字逐句的分析及解释。MTR是一份非常浓缩的文件，解释性的额外说明很少。因为文件中的每一句话都是有意义的，这份MTR精解的宗旨在于解析出隐藏于这些字句中的详细细节。它可以用作2级裁判的学习材料、为回归的裁判刷新知识、解决裁判间对问题的争论等，但它最主要的目的是教育。此文档的英文版位于[这里](https://blogs.magicjudges.org/rules/mtr/)，如果你发现问题、或想要提供建议，请发邮件给 Amtrfeedback@gmail.com。

AMTR的上一个汉化版本距今已有7年。本项目旨在对其进行翻新，并提供格式化的文本以及文本预处理工具，方便创作者构建术语库、翻译记忆库等资源，或与以往版本进行对比。

## 🧾 项目特点

* 对新版AMTR进行翻译。
* 出于术语对齐的目的，对MTR正文进行略微调整。
* 为了格式规范，对MTR进行清洗和排版，以提供干净的Markdown文件。
* 提供简易的文本预处理工具。

## ✨ 翻译说明与原则

规则内容讲究严谨性与一致性。前译者的工作相当优秀，我把没问题的段落用自己的话再翻译一遍只会造成混乱。已征得前译者杜昊老师的同意，本项目原则上与旧版相同的行为沿用旧版AMTR的翻译。

对于“上诉链”等在中文社区不流行的俗语进行了注释。对于AMTR原文中的部分错漏，在于原作者Bryan Prillaman交涉后进行了修改或注释。

对于MTR的译文中过于不讲究术语对齐的地方进行了修改。增添了MTR的附录A于2025年11月10日的更新。

## 🔧 构建方式

本项目提供两个自动化脚本，extract_parallel.py用于对源 Markdown 文档进行预处理以生成平行句对，extract_rules.py用于从发布AMTR的网站上提出所有注解，方便形成带翻译文件。

依赖库没几个，运行也很简单。不作赘述。
